COMOS Walkinside APIs
What's New in 10.2

This release introduces clear distinction between API support levels (see API Compatibility Policy), which allowed us to make a few experimental APIs (known as "preview" APIs), already available for early feedback.

Also, 10.2 is an "SP0" release, so it removes some older APIs that were already marked as deprecated in earlier releases.

Viewer SDK

Notable Additions

  • IVRViewerSdk has a new ForPreview property, which gives access to new "preview" APIs, namely Core SDK API and Scenario API.
  • Plug-ins can define a new field NeedsApplicationRestart, to tell Viewer if they require application restart for activation.
  • Settings have been redesigned and a number of new interfaces were added for settings access, like IVRSetting<T>. This is the preferred way to work with settings.

Notable Deprecations

  • Many old APIs for settings access were deprecated, especially those that exposed string names of settings. Please use new IVRSetting<T>-based APIs and do not assume knowledge of internal setting details, e.g. database names.
  • IVRBranchManager's highlights-related APIs were deprecated in favor of branch selection APIs (or IVRGroupManager for custom effects).
  • IVREventManager and related types were deprecated. Please do not use them and do not assume knowledge of internal details of event-passing protocol.
  • IVRVolumeManager and related types were deprecated. Please try new Core SDK capabilities or use alternative ways to visualize selection, like IVRGroupManager.
  • Many helpers were deprecated and duplicated in non-SDK assemblies WalkinsideUtil and WalkinsideUI. These include types like VRXMLFile and a few others.

Notable Removals

  • Many previously deprecated helper types were moved from Viewer SDK into non-SDK assemblies WalkinsideUtil and WalkinsideUI. These include types like VRTreeview, VRColorSlider, etc.

    If you depend on these types you may continue using them from those assemblies, but without compatibility guarantees of an API.

  • Previously deprecated IVRSelection was finally removed. Please use IVRGroupManager instead.
  • Previously deprecated IVRBranch.Type was finally removed. Please use IVRBranch.Kind and IVRBranch.Subtype instead.

Full details can be found in Viewer SDK API Reference.

Exporter SDK

Notable Changes

  • vrSettingsFileName/vrSettingsFileNameW now interprets its argument as output folder location of the exported model. Older behavior of this API was unpredictable and depended on whether the argument had extension.

    Example: if argument is C:\Spam\Bacon, then the .vrp will be created at C:\Spam\Bacon\Bacon.vrp.

Full details can be found in Exporter SDK API Reference.

Core SDK

Core SDK is a new "preview" API added in this release. It augments Viewer SDK API with sound and rendering capabilities. Same capabilities are used to power Walkinside ITS functionality.

Full details can be found in Core SDK API Reference.

Data SDK

Notable Additions

  • IProject has new methods AsStandaloneProject and AsOnlineProject, the latter providing access to new settings transfer functionality for online projects.

Full details can be found in Data SDK API Reference.

Virtual Room Client Library

Virtual Room Client Library is a new "preview" API added in this release. It is intended for developing Virtual Room agents that can observe other participants, drive avatars and communicate via custom messages.

Full details can be found in Virtual Room Client Library API Reference.
